# Development Diary - July 2026

---

## 2026-07-05: Start iOS original asset pipeline planning

Mapped the first pass of the iOS original asset replacement pipeline.

Added:
- `docs/WORKDIR/planning/PLAN-023_IOS_ORIGINAL_ASSET_PIPELINE.md`
- `docs/WORKDIR/planning/IOS_ORIGINAL_ASSET_MANIFEST_PLAN.md`
- `scripts/tooling/assets/scan_ios_asset_manifest.py`

The scanner inventories a staged deploy tree, classifies loose assets, reports
BIG archive containers, and emits both JSON and Markdown reports. Full asset
coverage still needs a BIG archive indexing/extraction backend because the
retail deploy stores most production assets in archives.

Follow-up in the same planning pass:
- Added BIG archive directory indexing for `BIGF` containers.
- Added archived text parsing for small INI/WND/MAP/STR/TXT-like entries.
- The manifest now separates indexed archive hits from hard missing references.
- Added first-pass typed dependency extraction for INI fields, W3D chunk/string
  references, binary map/UI tokens, and CSF label indexing.
- Wired iOS packaging to run manifest validation on bundled `GameData` before
  signing, failing the package when hard missing references remain.
- Added `build_ios_required_asset_manifest.sh` to reject incomplete fixture data
  and generate the hard manifest only from a staged tree with real BIG archives.
- Added `create_ios_playable_slice_worklist.py` to turn the hard manifest into
  an original-asset source/generated workspace and first-slice checklist.
- Added `GX_ORIGINAL_ASSET_PACK` packaging support so generated original
  `GameData` can be bundled and manifest-validated before signing.
- Added generated-slice validation against `playable_slice_worklist.json`, with
  package-time support through `GX_ORIGINAL_SLICE_WORKLIST`.
- Added `build_ios_original_asset_pack.py` and the first repo-owned generated
  iOS `GameData` pack. It bundles the WW3D runtime-required assets, the current
  WND boot UI fixture, iOS config, and a minimal English font binding so iPhone
  packaging can proceed with original non-retail resources.
- Updated iOS packaging to use `ios-original-assets/generated/GameData`
  automatically when no retail `GX_GAME_DATA` tree is available.
- Made iOS packaging more self-contained: it now builds the repo-owned original
  pack when needed, stages the renamed Liberation fonts automatically if they
  are absent, and reports that installs use bundled `GameData` instead of
  asking for a manual Documents copy.
- Expanded the generated iOS original pack with the mandatory startup INI
  surface (`Data/INI/Default/GameData.ini` and boot placeholders for the
  subsystem directories loaded during `GameEngine::init`) so the app no longer
  immediately depends on retail BIG archives to satisfy first-pass INI loads.
- Added generated boot WND layouts for the direct startup shell references
  (`Menus/BlankWindow.wnd`, `Menus/MainMenu.wnd`, and `Menus/LegalPage.wnd`)
  plus ShellMenuScheme INI placeholders.
- Expanded the boot INI placeholders to cover the next client/audio/UI startup
  directories loaded directly by the engine (`GameLOD`, audio/video groups,
  command buttons/sets, control-bar schemes, mouse, animation, particles,
  campaign, EVA, and related UI support).
- Added localized boot files for the iOS original pack (`Data/English/Language.ini`,
  `Data/English/HeaderTemplate`, and a minimal `Data/English/generals.csf`) and
  aligned the iOS runtime language/CSF casing so those bundled files are found
  on device.
- Added a single iPhone build/package entrypoint
  (`scripts/build/ios/build-package-ios-zh.sh`) that initializes the iOS DXVK
  source, fetches pinned MoltenVK, builds `ios-vulkan`, generates the
  repo-owned original asset pack, and hands off to the signed app packager.
- Expanded the generated original iOS pack with first-slice gameplay INI:
  playable/observer/civilian player templates, command buttons/sets, weapons,
  locomotors, armor, damage FX, five basic object templates, and generated
  command-bar icon textures.
- Wired the generated first-slice objects with real SAGE modules: W3D draw
  state, active/structure body health, unit AI and physics, locomotor sets, and
  production queues/exits for the command center and barracks.
- Replaced the zero-string iOS CSF placeholder with a deterministic generated
  CSF containing the playable-slice faction, object, and command text labels.
- Added a generated `IOSPlayableSlice` map, map cache entry, preview, and map
  string sidecars to the repo-owned iOS original pack. The iOS SDL entrypoint
  now auto-selects that map when no user `-initialFile` is provided, and the
  game engine starts it through an iOS-only skirmish setup with generated
  player/AI slots so starting units and buildings can be placed from the
  original `FactionIOS` template.
