# Development Diary - July 2026

---

## 2026-07-05: Start iOS original asset pipeline planning

Mapped the first pass of the iOS original asset replacement pipeline.

Added:
- `docs/WORKDIR/planning/PLAN-023_IOS_ORIGINAL_ASSET_PIPELINE.md`
- `docs/WORKDIR/planning/IOS_ORIGINAL_ASSET_MANIFEST_PLAN.md`
- `scripts/tooling/assets/scan_ios_asset_manifest.py`

The scanner inventories a staged deploy tree, classifies loose assets, reports
BIG archive containers, and emits both JSON and Markdown reports. Full asset
coverage still needs a BIG archive indexing/extraction backend because the
retail deploy stores most production assets in archives.

Follow-up in the same planning pass:
- Added BIG archive directory indexing for `BIGF` containers.
- Added archived text parsing for small INI/WND/MAP/STR/TXT-like entries.
- The manifest now separates indexed archive hits from hard missing references.
- Added first-pass typed dependency extraction for INI fields, W3D chunk/string
  references, binary map/UI tokens, and CSF label indexing.
- Wired iOS packaging to run manifest validation on bundled `GameData` before
  signing, failing the package when hard missing references remain.
- Added `build_ios_required_asset_manifest.sh` to reject incomplete fixture data
  and generate the hard manifest only from a staged tree with real BIG archives.
- Added `create_ios_playable_slice_worklist.py` to turn the hard manifest into
  an original-asset source/generated workspace and first-slice checklist.
- Added `GX_ORIGINAL_ASSET_PACK` packaging support so generated original
  `GameData` can be bundled and manifest-validated before signing.
- Added generated-slice validation against `playable_slice_worklist.json`, with
  package-time support through `GX_ORIGINAL_SLICE_WORKLIST`.
- Added `build_ios_original_asset_pack.py` and the first repo-owned generated
  iOS `GameData` pack. It bundles the WW3D runtime-required assets, the current
  WND boot UI fixture, iOS config, and a minimal English font binding so iPhone
  packaging can proceed with original non-retail resources.
- Updated iOS packaging to use `ios-original-assets/generated/GameData`
  automatically when no retail `GX_GAME_DATA` tree is available.
- Made iOS packaging more self-contained: it now builds the repo-owned original
  pack when needed, stages the renamed Liberation fonts automatically if they
  are absent, and reports that installs use bundled `GameData` instead of
  asking for a manual Documents copy.
- Expanded the generated iOS original pack with the mandatory startup INI
  surface (`Data/INI/Default/GameData.ini` and boot placeholders for the
  subsystem directories loaded during `GameEngine::init`) so the app no longer
  immediately depends on retail BIG archives to satisfy first-pass INI loads.
- Added generated boot WND layouts for the direct startup shell references
  (`Menus/BlankWindow.wnd`, `Menus/MainMenu.wnd`, and `Menus/LegalPage.wnd`)
  plus ShellMenuScheme INI placeholders.
- Expanded the boot INI placeholders to cover the next client/audio/UI startup
  directories loaded directly by the engine (`GameLOD`, audio/video groups,
  command buttons/sets, control-bar schemes, mouse, animation, particles,
  campaign, EVA, and related UI support).
- Added localized boot files for the iOS original pack (`Data/English/Language.ini`,
  `Data/English/HeaderTemplate`, and a minimal `Data/English/generals.csf`) and
  aligned the iOS runtime language/CSF casing so those bundled files are found
  on device.
- Added a single iPhone build/package entrypoint
  (`scripts/build/ios/build-package-ios-zh.sh`) that initializes the iOS DXVK
  source, fetches pinned MoltenVK, builds `ios-vulkan`, generates the
  repo-owned original asset pack, and hands off to the signed app packager.
- Expanded the generated original iOS pack with first-slice gameplay INI:
  playable/observer/civilian player templates, command buttons/sets, weapons,
  locomotors, armor, damage FX, five basic object templates, and generated
  command-bar icon textures.
- Wired the generated first-slice objects with real SAGE modules: W3D draw
  state, active/structure body health, unit AI and physics, locomotor sets, and
  production queues/exits for the command center and barracks.
- Replaced the zero-string iOS CSF placeholder with a deterministic generated
  CSF containing the playable-slice faction, object, and command text labels.
- Added a generated `IOSPlayableSlice` map, map cache entry, preview, and map
  string sidecars to the repo-owned iOS original pack. The iOS SDL entrypoint
  now auto-selects that map when no user `-initialFile` is provided, and the
  game engine starts it through an iOS-only skirmish setup with generated
  player/AI slots so starting units and buildings can be placed from the
  original `FactionIOS` template.
- Added generated in-game WND layouts for the direct gameplay path
  (`ControlBarPopupDescription`, science points, chat/MOTD, hidden control bar,
  score, communicator, victory/defeat, observer quit, and CRC mismatch) so the
  iOS original pack covers the shell-to-match UI transition instead of stopping
  at boot menus.
- Added an `IOSDozer` builder unit to the generated playable slice, including
  command center production, a starting-unit slot, `DozerAIUpdate`, structure
  build commands, and a generated command-bar icon so the iOS faction can place
  its power plant and barracks instead of only spawning combat units.
- Replaced direct `ShatterPlanes0` object model references with generated
  object-specific W3D lookup names (`IOSCommandCenter`, `IOSPowerPlant`,
  `IOSBarracks`, `IOSDozer`, `IOSRanger`, and `IOSScoutVehicle`) and staged
  matching W3D files in both root and runtime-required lookup locations. This
  keeps the slice on valid WW3D assets while removing the shared placeholder
  model dependency from gameplay INI.
- Added the direct in-game UI WND layouts created by `InGameUI`
  (`ControlBar.wnd`, `ReplayControl.wnd`, and `InGamePopupMessage.wnd`) to the
  generated iOS pack so the match runtime has its expected HUD and popup layout
  files in addition to the earlier shell/gameplay popups.
- Added generated load-screen layouts for the direct `LoadScreen` paths and a
  compact set of generated root/`Data/Art/Textures` runtime textures used by
  hardcoded W3D terrain, shroud, water, shadow, bib, waypoint, scorch, and
  missing-texture lookups. These keep the generated iOS pack from falling
  through to absent retail art during match startup and first gameplay render.
- Fixed the iOS native-resolution startup path: the previous code appended
  synthetic `-xres/-yres` arguments after command-line parsing had already run,
  so the override never reached the engine. The SDL entrypoint now writes the
  native drawable size directly into `GlobalData` unless the user explicitly
  supplied resolution flags.
- Tightened iOS packaging so the lower-level app packager follows
  `GX_IOS_PRESET`, uses the same `GX_MOLTENVK_ROOT` default as the MoltenVK
  fetch script, and creates the manifest directory before optional generated
  slice validation. This keeps the one-command package/install path aligned
  with the generated original asset flow.
- Added generated modal/session WND layouts for direct gameplay-adjacent paths:
  message boxes, quit flows, options, save/replay popups, and disconnect screen.
  This keeps runtime error/exit/session handling inside the self-contained iOS
  pack instead of depending on retail UI assets.
- Replaced the generated `ControlBar.wnd` blank placeholder with a compact
  playable HUD layout containing the exact runtime window names used by
  `ControlBar` and `InGameUI`: control-bar parent, command buttons, production
  queue slots, economy/power widgets, right-HUD cameo/upgrade slots, beacon,
  construction/timer panels, radar input area, and options/communicator buttons.
  This removes a direct null-dereference risk during match HUD initialization
  and gives generated command sets real button windows to bind on iOS.
- Replaced the generated `GeneralsExpPoints.wnd` and
  `ControlBarPopupDescription.wnd` placeholders with named runtime layouts for
  the science purchase panel and command tooltip popup. The generated pack now
  includes rank 1/3/8 science buttons, the panel close button, experience text
  and bar widgets, and tooltip name/cost/description fields, matching the
  direct control-bar lookups used during gameplay.
- Replaced additional in-match placeholder layouts with named generated WNDs
  for chat, popup messages, replay controls, and MOTD handling. The generated
  iOS pack now includes `InGameChat.wnd:TextEntryChat`,
  `InGameChat.wnd:StaticTextChatType`, `InGameChat.wnd:ButtonClear`,
  `InGamePopupMessage.wnd:InGamePopupMessageParent`,
  `InGamePopupMessage.wnd:StaticTextMessage`, `InGamePopupMessage.wnd:ButtonOk`,
  `ReplayControl.wnd:ParentReplayControl`, and `MOTD.wnd:MOTD`, with the
  matching callback names wired into the generated layouts.
- Replaced the generated single-player and shell load-screen placeholders with
  named runtime layouts. The generated iOS pack now includes the
  `SinglePlayerLoadScreen` parent, progress bar, percent label, objective
  window, five objective text lines, cameo/head windows, four cameo/location
  labels, plus `ShellGameLoadScreen` progress and legal text fields, matching
  the assert-checked load-screen lookups used while entering the generated map.
- Replaced the generated main-menu placeholder with a named shell layout for
  the direct `MainMenuInit` and W3D main-menu lookup paths. The generated pack
  now includes `MainMenuParent`, core shell buttons, all five dropdown
  `MapBorder` windows, faction/grow marker windows, version/ruler text, and the
  main-menu callbacks, preventing the shell fallback path from dereferencing
  absent placeholder widgets.
- Added an iOS startup GameData probe after switching into
  `<GeneralsXZH.app>/GameData`. Device logs now explicitly report whether the
  required generated pack files are readable from the signed app bundle before
  the engine initializes.
- Replaced generated audio INI placeholders with a bounded iOS audio profile:
  low sample/stream counts, a 2 MiB cache cap, silent default sound/music/dialog
  entries, silent definitions for common generated-slice events
  (`PlaceBuilding`, `RallyPointSet`, `LoadScreenAmbient`, etc.), and `MiscAudio`
  bindings to `NoSound`. This keeps OpenAL initialization and gameplay audio
  event dispatch from chasing missing retail files while preserving the engine's
  normal audio code path.
- Added a tiny real generated-slice sound set from the user-provided Octave
  sounds. Five short mono 44.1 kHz WAVs are now staged from
  `ios-original-assets/source/audio/octave/` into `Data/Audio/Sounds` and wired
  to UI click, placement, rally-point, rejection, and load ambient events. The
  Octave license is stored next to the source WAVs.
